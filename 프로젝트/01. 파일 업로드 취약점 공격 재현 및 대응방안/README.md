# WebShell 기반 파일 업로드 취약점을 이용한 권한 상승과 Database 탈취 및 대응방안

- 선정 이유 : 최근 웹 서비스는 게시판, 파일 첨부, 이미지 업로드 등 사용자 입력 기반 기능을 기본적으로 포함하고 있습니다.  
  그중 파일 업로드 기능은 구현 난이도가 낮지만 보안 검증이 미흡할 경우 서버 침해로 직결될 수 있는 대표적인 공격 지점입니다.   
  단순 확장자 필터링만으로는 충분하지 않으며, 실제 침해 사례에서도 파일 업로드 취약점을 시작으로   
  시스템 전체가 침해되는 경우가 반복적으로 발생하고 있습니다.   
  이에 따라 단일 취약점이 어떻게 연쇄적인 침해로 확장되는지를 구조적으로 이해하고자 본 주제를 선정하였습니다.

---

## 공격 시나리오 
- 입력값 검증 순서의 오류를 이용해 파일 업로드 필터를 우회하고 웹쉘을 업로드한 뒤, 서버 내부 정보를 수집하여 데이터베이스 계정 탈취 및 권한 상승을 수행합니다

---

## 문제 원인 및 간단한 공격 결과
- 입력값 정규화와 검증 처리 순서가 잘못 설계되어 필터링 이후 디코딩 과정에서 보안 검증이 우회되도록 구현된 것이 문제의 원인입니다.
- 공격 결과 서버내의 Database 정보 확인 및 테이블 컬럼 유출, SUID 권한 남용으로 인한 일반사용자->루트권한이용 의 권한 상승이 가능합니다.
